> Using Local Methods in Classes
July 6, 2014

This week in DBC we had our formal introduction to classes. We saw them last week in one of our challenges and we did some work with them (again, without the formal introduction) when learning JavaScript. Learning classes has greatly expanded my toolset and my ability to tackle more complex challenges.

The beauty of Classes is that it allows you to treat variables as complex objects with multiple properties and capabilities. It is rare that one variable would have the ability to take one action (or method) and that's it. With local methods within classes, we can assign and combine various actions to an object. While we can build some complexity into our code without classes, it often requires using a number of nested if/else statements, whereas the use of multiple methods within one class allows us to write cleaner code that can be broken apart more easily.

My background is in data analysis in the education sector, so this is where I'll draw from for this example. When analyzing data, it is rare that I would ever perform just one calculation on a dataset. It is also rare that I could use my dataset it its raw form. Normally, I would need to perform a transformation on the data so that I could begin to work with it, and then I would run my analysis. When working with student data, especially their attendance and grades, this information was constantly changing, so any analysis would require me to download my data-set in real-time, transform it, and then perform some analysis based on what was needed or requested at the given moment.

Let's run through an example using student attendance data. I'm going to super-simplify this, just to show the different ways methods can be used with classes. In this example, I run a few different sample analyses to calculate quarter, semester, and annual attendance rates, and to compare quarterly rates.
