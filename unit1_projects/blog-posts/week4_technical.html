> Enumberables: Each v. Map
June 29, 2014

As a novice Ruby programmer-in-training, I'm challenged to write new methods every week. This often involves research difference pieces of my potential solution to understand how I might arrive at my desired end result. This week in my research, I came across #map. At first glance, it seemed to basically be the same as #each. Upon a closer look, there is a key difference between the two.


Each v. Map

The essential difference between each and map is that map will actually alter an array, wheras each requires each does not. Both #each and #map only function within the block in which their being used. Therefore, even though #map may make changes to the array, the changes are only within to the block within that method.

Block is used to ensure that content appears on the next line. Dimensions (width and height) can be specified. Divs are by default set to disply block.

Example code

First, define an array:

array = [1,2,3]
=> [1, 2, 3]

Next, lets define a method using each

def add_each(array)
array.each {|x| x+2}
end

add_each(array)
=> [1, 2, 3]

Let's check on our array. What is it defined as?

array
=> [1, 2, 3]

Now, lets define a method using each, but this time use puts

def add_each(array)
array.each {|x| puts x+2}
end

add_each(array)
=> 3
4
5
[1, 2, 3]

You see the method is returning the value of each of the numbers in our original array +2. However, it is still showing that array = [1,2,3]

array
=> [1, 2, 3]

Last, lets define a method using #map. We will use the exact same format as our first #each method.

def add_map(array)
array.map {|x| x+2}
end

add_map(array)
=> [3, 4, 5]

Now, the method is returning the array with the numbers +2. Let's check on our array, though. Will it return array = [3,4,5]?

array
=> [1, 2, 3]

Nope! That's because #each and #map are only working on the blocks within the method that they are being run. The changes are not to the value of the variable itself outside of the method.
