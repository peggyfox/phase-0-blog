Github

Definitions

  • add
    add file contents to the index. This stages a file to be committed, but does not actually commit it.
    • branch
    list, create, or delete branches.
  • checkout
    checkout and swith to a branch. This can be used to revert a file to its most recent commited state.
  • clone
    clone a repository into a new directory: basically copy and paste
  • commit
    record changes to the repository. This makes a new "blob" of your file - a new version that you will be able to refer back to.
  • fetch
    download changes from another repository and merge them with the branched repository on yoru computer.
  • log
    show commit logs, like an activity log
  • merge
    join two or more development histories - allows combination of 2+ versions
  • pull
    fetch from and merge with another repository or a local branch.
  • push
    update remote refs along with associated objects. like git fetch and git merge together
  • reset
    reset most recent commit to the specified state
  • rm
    remove files from the working tree and the index.
  • status
    show the working tree status

Step-by-Steps

Push files to a remote repository

  1. Navigate to branch to be committed.
  2. Git Status
  3. Add files to stage them to be committed "git add *.html" (to add everything of the same file type")
  4. "git commit -m "note""
  5. "git push" - if pushing for the first time type "git -u origin master"

Fetch changes

  1. Navigate to the master branch (i.e. phase_0_unit_1). Check to make sure it is the master by typing git branch in the terminal
  2. check the current remote. type "git remote -v"
  3. Fetch the changes. type "git fetch upstream". You can check that the changes are stored on your local branch by typing "git branch -va"
  4. Merge the changes with your master branch. type "git merge upstream/master"
  5. Resolve any conflicts with "git status" or "git diff"
  6. Commit changes: "git commit -m "commit one""

Commit locally
Do everything from Push files to a remote repository, just skip the last step where you'd push to the repository.
